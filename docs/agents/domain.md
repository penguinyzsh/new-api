# 领域文档

本文件规定工程技能在探索代码库时如何读取本仓库的领域文档。

## 探索前需要阅读

- 根目录中的 **`CONTEXT.md`**
- 根目录中的 **`docs/adr/`**，并阅读与即将处理区域相关的 ADR

如果这些文件或目录不存在，直接继续，不报告缺失，也不主动建议预先创建。`/domain-modeling` 技能会在术语或决策真正明确后按需创建；该技能可由 `/grill-with-docs` 和 `/improve-codebase-architecture` 触发。

## 文件结构

本仓库采用单上下文布局：

```text
/
├── CONTEXT.md
├── docs/adr/
│   ├── 0001-示例决策.md
│   └── 0002-另一项示例决策.md
└── ...
```

## 使用术语表中的词汇

输出中命名领域概念时，包括 Issue 标题、重构建议、假设和测试名称，必须使用 `CONTEXT.md` 中定义的术语，不得改用术语表明确排除的同义词。

如果术语表中没有所需概念，应先判断是否使用了项目并不采用的语言；若确实存在领域语言缺口，则记录下来，交由 `/domain-modeling` 处理。

## 明确指出与 ADR 的冲突

如果输出与现有 ADR 冲突，必须明确指出，不得静默覆盖。例如：

> 与 ADR-0007（事件溯源订单）冲突，但由于……值得重新讨论
